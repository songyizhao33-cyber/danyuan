# 旦缘 - 复旦大学学生恋爱匹配平台技术架构

## 1. 系统架构

### 1.1 整体架构
- **前端**：React + TypeScript + Tailwind CSS
- **后端**：Node.js + Express
- **数据库**：MongoDB
- **认证**：JWT
- **邮件服务**：Nodemailer
- **部署**：Vercel (前端) + Railway (后端) + MongoDB Atlas (数据库)

### 1.2 核心组件
1. **用户认证模块**
   - 学邮验证（后缀为edu.cn）
   - 验证码发送与验证
   - 密码设置与管理
   - JWT token生成与验证

2. **问卷系统模块**
   - 五大问卷部分：基础信息、家庭婚育态度、生活习惯、兴趣爱好、人格测试
   - 自动保存功能（翻页自动保存）
   - 草稿管理
   - 问卷进度跟踪

3. **匹配算法模块**
   - 基于变量空间的聚类算法
   - 定期执行匹配任务
   - 匹配结果存储与管理

4. **邮件通知模块**
   - 验证码发送
   - 匹配结果通知
   - 系统通知

5. **管理员面板**
   - 用户数据管理
   - 匹配统计
   - 系统监控

6. **辅助功能模块**
   - 匹配倒计时
   - 参与人数显示
   - 系统状态展示

## 2. 技术栈选择

### 2.1 前端技术
- **React 18**：构建用户界面
- **TypeScript**：类型安全
- **Tailwind CSS**：响应式设计
- **React Router**：路由管理
- **React Hook Form**：表单管理
- **Zod**：数据验证
- **Framer Motion**：动画效果

### 2.2 后端技术
- **Node.js 18+**：运行环境
- **Express**：Web框架
- **MongoDB**：数据库
- **Mongoose**：ODM
- **JWT**：认证
- **Nodemailer**：邮件发送
- **Cron**：定时任务

### 2.3 数据库设计
- **users**：用户信息
- **questionnaires**：问卷数据
- **matches**：匹配结果
- **admin**：管理员信息

### 2.4 安全考虑
- HTTPS加密
- 密码哈希存储
- 验证码安全
- 防SQL注入
- 防XSS攻击

## 3. 系统流程

### 3.1 用户注册流程
1. 用户输入学邮（验证后缀为edu.cn）
2. 系统发送验证码到学邮
3. 用户输入验证码验证
4. 设置密码
5. 完成注册

### 3.2 问卷填写流程
1. 用户登录后进入问卷页面
2. 填写问卷（五大部分）
3. 翻页时自动保存
4. 完成问卷后提交
5. 系统将用户加入匹配池

### 3.3 匹配流程
1. 系统每周定时执行匹配算法
2. 基于用户问卷数据进行聚类分析
3. 生成匹配结果
4. 通过邮件通知用户匹配结果

### 3.4 管理员流程
1. 管理员登录后台
2. 查看用户数据和匹配统计
3. 管理系统设置
4. 监控系统运行状态
